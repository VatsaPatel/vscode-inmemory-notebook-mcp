Notebook MCP Tool Guide

Current compact Python-focused surface. Connect once to the global daemon URL, then pass notebook_uri on every notebook tool call.

Default Workflow

Explore

Call notebook_status, choose a notebook_uri, then call notebook_read. The read response includes source, outputs, lock state, and cell_id anchors.

Fallback ids like index:3 are only stable until cells are inserted, deleted, or moved.

Edit

Use notebook_edit_cells with before_cell_id or after_cell_id. Avoid raw indexes when an anchor is available.

Run Long Jobs

For Spark/Python jobs, insert a visible cell, call notebook_run with a short wait_ms, then poll notebook_operation.

Protect Work

Use notebook_lock_cell for human-protected cells. Generic metadata mutation is not part of the simplified surface.

Tool Surface

ToolPurpose
notebook_helpUse when unsure; returns short workflow recipes.
notebook_statusUse first; shows daemon state and open notebook URIs.
notebook_open / notebook_createUse when the notebook is not open, or a fresh Python notebook is needed.
notebook_readUse first for notebook context or selected cells. No structured outline; re-read after reorders.
notebook_searchUse to find text or replace with dry-run first.
notebook_edit_cellsUse for insert/update/delete/type changes; prefer cell_id anchors.
notebook_move_cellsUse to reorder cells as a block; call notebook_read afterward.
notebook_clear_outputsUse to clear one cell or the whole notebook.
notebook_lock_cellUse to protect/unprotect agent-edit access. Not a general metadata tool.
notebook_save / notebook_exportUse to persist notebook work or produce markdown/Python/HTML output.
notebook_runUse for visible cell/range/all execution; scope:"code" is only for throwaway probes.
notebook_operationUse to list/get/wait/stream outputs. It never cancels.
notebook_cancel_executionUse only to stop a running operation.
notebook_get_kernel_infoUse for Python kernel state and best-effort context; prefer visible cells for variable inspection.
notebook_kernel_controlUse only to restart or interrupt the kernel.

Examples

Read Then Insert

{
  "tool": "notebook_read",
  "arguments": { "notebook_uri": "file:///path/to/notebook.ipynb", "include_outputs": "summary", "response_format": "json" }
}
{
  "tool": "notebook_edit_cells",
  "arguments": {
    "operations": [{
      "op": "insert",
      "notebook_uri": "file:///path/to/notebook.ipynb",
      "after_cell_id": "index:3",
      "cells": [{ "type": "code", "content": "df.explain('formatted')" }]
    }],
    "response_format": "json"
  }
}

Run And Stream

{
  "tool": "notebook_run",
  "arguments": { "notebook_uri": "file:///path/to/notebook.ipynb", "scope": "cell", "cell_id": "index:4", "wait_ms": 1000, "response_format": "json" }
}
{
  "tool": "notebook_operation",
  "arguments": { "operation_id": "op_...", "wait_ms": 0, "include_partial": true, "response_format": "json" }
}
